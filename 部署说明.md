# 飞行家 · AI 旅行规划 —— 部署说明（保姆级）

发布后，别人拿到你的网址就能直接用，**看不到你的 API Key**（Key 藏在服务器）。全程免费。

## 项目文件

```
travel-app/
├── index.html        网页界面（用户看到的）
├── api/
│   └── chat.js       服务器中转（把 Key 藏在这里，用户看不到）
├── package.json      项目配置
└── 部署说明.md        本文件
```

---

## 一、先准备两个免费账号

1. **GitHub**：打开 github.com 注册（用来存代码）。
2. **Vercel**：打开 vercel.com，点 “Sign Up”，选 “Continue with GitHub” 用刚才的 GitHub 登录（用来发布网站）。

---

## 二、把代码传到 GitHub（网页操作，不用命令行）

1. 登录 GitHub，右上角 “+” → **New repository**（新建仓库）。
2. 仓库名随便填，比如 `travel-app`，选 **Public** 或 **Private** 都行，点 **Create repository**。
3. 进入空仓库页面，点 **uploading an existing file**（上传现有文件）。
4. 把本文件夹里的 **index.html、package.json、部署说明.md** 三个文件，以及 **api 文件夹**一起拖进去上传。
   - 注意：`api/chat.js` 要保持在 `api` 文件夹里。直接把整个 `api` 文件夹拖进去即可。
5. 点最下方 **Commit changes** 完成上传。

---

## 三、用 Vercel 一键发布

1. 登录 vercel.com，点 **Add New… → Project**。
2. 找到刚才的 `travel-app` 仓库，点 **Import**。
3. **关键一步**：在 **Environment Variables（环境变量）** 里添加你的 Key：
   - Name（名称）：`SILICONFLOW_API_KEY`
   - Value（值）：粘贴你的硅基流动 Key（`sk-...`）
   - 点 **Add** 添加。
   - （可选）再加一个 `MODEL`，值填 `deepseek-ai/DeepSeek-V3`，不加就用默认。
4. 点 **Deploy**，等 1 分钟左右。
5. 出现庆祝画面后，点 **Visit / 你的域名**（形如 `https://travel-app-xxx.vercel.app`），这就是**可以发给任何人的链接**。

---

## 四、（推荐）发布后加一道来源保护

防止别人把你的接口地址拿去别的网站盗用你的额度：

1. 复制你 Vercel 分配的网址，比如 `https://travel-app-xxx.vercel.app`。
2. Vercel 项目 → **Settings → Environment Variables**，新增：
   - Name：`ALLOWED_ORIGIN`
   - Value：你的网址（不要带最后的斜杠），如 `https://travel-app-xxx.vercel.app`
3. 保存后到 **Deployments** 页，对最新记录点 **Redeploy** 让它生效。

---

## 五、务必设一个花费上限（防止被刷）

代码里已内置：限制单次输出长度、每 IP 每分钟最多 5 次、锁定只能做旅行规划。但最稳的一道防线是账户层面：

- 登录**硅基流动**控制台，只充少量余额（比如 20–50 元），并留意用量。
- 余额用完接口会自动停，不会欠费，等于天然的“熔断”。

---

## 常见问题

- **点生成后报“服务器未配置 API Key”**：环境变量 `SILICONFLOW_API_KEY` 没加或拼错，去 Vercel Settings 里检查，改完记得 Redeploy。
- **报模型不存在**：`MODEL` 要用带前缀的完整名，如 `deepseek-ai/DeepSeek-V3`，以硅基“模型广场”页面显示的为准。
- **想换域名**：Vercel 项目 Settings → Domains 里可绑定自己的域名。
- **改了代码怎么更新**：在 GitHub 上重新上传文件，Vercel 会自动重新部署。
